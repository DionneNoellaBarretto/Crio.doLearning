Summary :

Basic workflow in Git

  • You modify files in your working directory
  • You stage the files, adding snapshots of them to your staging area (git add file)
  • You do a commit, which takes the files as they are in the staging area and stores that snapshot permanently to your local Git repository (git commit file)
  • You push the changes from the local repository to the remote repository (git push)
  • You pull the changes from the remote repository to the local repository (git pull)

Summary :
Web development - includes client-side and server-side development
Frontend development (client-side)
  • HTML - Structure
  • CSS - Style
  • JS - Functionality
Browser (client) communicates with the server on your behalf
  • HTTP is the language of communication
  • Browser sends a request each for any extra information/resource it needs
  1. HTML is used to describe the contents of your webpage
  2. All HTML content goes inside the "htm" tag.
  3. HTML document's metadata goes inside the "head" tag and all the content to be displayed inside the "body" tag
  4. There are different HTML tags to perform different actions
  5. "b" - bolds text
  6. "p" - wraps text as a paragraph
  7. "a" - add hyperlink
  8. CSS makes the HTML content look beautiful
  9. CSS can be added to the HTML
  10. Externally - linking css files
  11. Internally - using "style" tag
  12. Inline - using style attribute
  13. CSS flexbox makes it easier to organize a group of elements
  14. CSS positioning can be used to set the position of an HTML element in the web page. For example, you can fix an element to the top of the web page even when scrolling.
  15. Dead links are needed often when you want to add anchor elements to your page before you know where they will point to. Remember that dead links always reload the page! To work around this link to # as a placeholder value within "" of the href
  16. Lists can be added using ul, ol tags with each item under li tag
  17. Each HTML tag can have multiple attributes
  18. Anchor tags are used for links, target attribute to define where to open the link
  19. Links can be relative or absolute
  20. Absolute links (or urls) ⇒ Full Web Address Example: "https://resuminator.in", "https://crio.do"
  21. Relative URL ⇒ A link to page within the website i.e. without the https:// or www part
    22. Example: "/home", "/about"
  22. We can also link files like images or other pages using the relative URLs
    23. Example: "about/team.html" → Links to the team.html file in the about folder.
  23. Id attribute uniquely identifies an HTML Element
  24. # can be used in anchor tag href to link to a section using its id
  25. Images in HTML use the "img" tag with src, alt, width, and height attribute

Summary :
  • Place bootstrap link before any other stylesheet ensures Bootstrap rules get prioritized if there’s a conflict between the styles in Bootstrap CSS and the CSS file that you might have created. Find more here .
  • The collapse class hides the content of its
    element (links) and navbar-collapse adds functionality to toggle display of the links when the toggle icon is clicked.
  • class="navbar-toggler" - works along with the collapse class to hide an element and show the toggle button instead
  • type="button" - creates a button with no default behavior when clicked. See other types here
  • data-bs-toggle="collapse" and data-bs-target="#collapsibleNavbar" - makes content of the "div class="collapse" id="collapsibleNavbar"" element collapse to the button on smaller screens.
  • Cards in Bootstrap
    1. card class is the root class for a card
    2. card-body wraps the body of the card. If header or footer is present they’d come outside the card-body 'div'
    3. card-text applies styles to the text
    4. btn sets the hyperlink to look like a button and btn-primary sets the color. Feel free to use other colors like btn-success, btn-secondary.
  • "Bootstrap’s grid system uses a series of containers, rows, and columns(12) to layout and align content. It’s built with flexbox and is fully responsive." - Bootstrap Docs
  • Spacing (Margin/Padding)
    1. mt-0 - sets margin on top to 0
    2. mx-5 - sets margin on x-axis i.e, both left and right to size 5 which translates to 3rem units
    3. p-0 - sets padding on all 4 sides to 0
    4. pr-md-0 - sets padding on right for "md" sized and larger devices to 0
